## Explicit Limitations

- The Sepolia `MockUSDT` placeholder address has no bytecode and is marked `missing-contract`.
- WDK fee quote for the placeholder transaction may be unsupported because the ephemeral proof account is unfunded and the placeholder is not a functional ERC-20 contract.
- The one-time consumption store is in-memory and concurrency-safe only inside one Node process; production requires durable transactional storage.
- No smart-contract enforcement is implemented in this branch.
- No transaction is broadcast, and the browser does not execute native WDK wallet operations.
